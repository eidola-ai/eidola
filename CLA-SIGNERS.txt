# CLA Signers
#
# This file, together with the relevant Git history, is the signature ledger
# for the CLA flow used by this repository.
#
# Format: <cla-hash> <git-email-pattern> <signer-email> <signer-name> [| <corporation-name>]
#
# - cla-hash:          SHA-256 of the CLA text (CLA-INDIVIDUAL.md or CLA-CORPORATE.md)
#                      This binds the signer to an exact version of the agreement.
# - git-email-pattern: Exact email for individuals, @domain for corporate signers
# - signer-email:      The email of the person who signed
# - signer-name:       Full legal name of the signer
# - corporation-name:  (Corporate CLA only) Full legal name of the Corporation, 
#                      separated by a pipe | from the signer name.
#
# If either CLA markdown file changes, its hash changes and a new signer entry is
# required for that new version.
#
# Individual example:
#   abc123... alice@example.com alice@example.com Alice Smith
#
# Corporate example (covers all @acme.com committers):
#   def456... @acme.com jane@acme.com Jane Doe | Acme Corp

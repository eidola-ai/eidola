## How To Sign

1. Compute the SHA-256 hash of this file.
2. Add a line to `CLA-SIGNERS.txt` in this format:

   `<cla-hash> <git-email-pattern> <signer-email> <signer-name> | <corporation-name>`

3. For a corporate CLA, set `git-email-pattern` to a covered email domain (e.g.,
   `@example.com`). This must start with the `@` symbol. Contributions whose Git
   author or committer email matches that domain are treated as covered by this
   Agreement.
4. Commit that change to this repository.

The line in `CLA-SIGNERS.txt` and the relevant Git history for the commit that
adds it are the signature record for this Agreement. No PDF, email, or
handwritten signature is required.
